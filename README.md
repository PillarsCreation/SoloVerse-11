# FloodGuard 洪水安防地理智能系统

山洪/山体滑坡预警、实时救援调度、灾后农业损失统计一体化智能系统。

## 项目概述

FloodGuard 是一套面向山区山洪、山体滑坡灾害的全栈Web+移动端自适应智能预警调度系统，涵盖**灾害推演预警、实时救援调度、灾后农业损失统计**三大业务闭环。系统对标外卖平台三方协同模式（管理后台-用户-骑手），实现从"单一报警"到"精准推演行动指挥"的升级。

### 核心特性

- **完整山洪预警模型链**：八环节推演（降雨输入→产汇流→沟道演进→淹没分析→风险评估→分级预警→转移任务→反馈校正），每条预警附带完整可溯源依据
- **GIS 地图可视化**：基于 Leaflet 的地理空间展示，风险区/人员/安置点/救援轨迹全坐标化
- **四类角色权限隔离**：管理端、村民端、救援端、统计员端，各司其职、数据互通
- **外卖式救援追踪**：实时轨迹、ETA、进度时间线
- **国标四级预警**：红/橙/黄/蓝，高风险触发声光+语音强制告警
- **移动端+PC端全适配**：村民/救援端移动优先，管理/统计员端PC大屏

## 四类角色功能

### 角色1：水利气象调度管理端（PC 1920×1080）

- **指挥调度大屏**：左220px菜单 + 中间GIS地图 + 右320px数据台账
- **预警发布中心**：八环节模型链推演、双文案（村民通俗版+专业溯源版）、四级发布
- **人员GIS台账**：全域人员坐标、状态、伤情、物资需求
- **救援全局调度**：任务创建、队伍指派、轨迹监控
- **灾情数据汇总**：损失统计、物资缺口、按村汇总

### 角色2：辖区村民用户端（移动端 375×667）

- **强制预警推送**：弹窗+语音播报，红橙高风险强制告警
- **标准化避险指引**：山洪/山体滑坡官方逃生教程
- **安置点信息登记**：GPS坐标自动上传、同行人数、拍照
- **伤情与物资上报**：三级伤情+五类物资需求
- **外卖式救援追踪**：实时轨迹、ETA、进度时间线

### 角色3：应急救援队伍端（移动端）

- **任务自动接收**：目标坐标、危险提示、最优路线
- **自动轨迹上报**：每30秒GPS定位上报
- **现场处置更新**：出发→抵达签到→完成任务
- **离线路线导航**：规避高危路段

### 角色4：农业灾情统计员端（PC）

- **灾情登记表单**：种植业+养殖业损失录入
- **灾情台账**：分页表格、CSV导出
- **汇总统计**：按村组汇总、可视化展示

## 山洪预警模型链

系统实现了完整的八环节推演引擎：

| 环节 | 名称 | 核心算法 |
|------|------|----------|
| 1 | 降雨输入 | 多源数据融合（雷达+站点） |
| 2 | 产汇流计算 | SCS-CN曲线数法 |
| 3 | 沟道演进 | 马斯京根法简化 |
| 4 | 淹没分析 | 基于地形坡度+流速估算 |
| 5 | 风险评估 | 危险性×暴露度×脆弱性综合评判 |
| 6 | 分级预警 | 国标红/橙/黄/蓝四级 |
| 7 | 转移任务 | 自动生成转移清单+路线+安置点 |
| 8 | 反馈校正 | 预留参数校正接口 |

每条预警的 `derivation` 字段完整存储上述推演结果，确保可溯源、可核查。

## 核心交互流程

```
管理端发布预警 → 村民端接收强制告警
→ 村民避险登记+上报伤情物资
→ 管理端生成救援任务+指派队伍
→ 救援队伍出动+自动轨迹上报
→ 村民端实时追踪+管理端全局监控
→ 救援完成闭环
→ 灾害结束→统计员录入灾后损失
```
